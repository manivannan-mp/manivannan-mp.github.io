I taught Claude to stop making me repeat myself — and it gave me my evenings back

Most of us use Claude for one-off tasks. There's a second pattern that quietly eats hours every week — and a small skill that fixes it.

Draft this, review that, summarize this. That's how most of us use Claude day to day — one-off tasks, one at a time. But there's a second pattern that quietly eats hours every week: the recurring check. The daily pipeline scan. The weekly progress report. The nightly log review. Work that isn't hard, just relentless.

I built a Claude Code skill called build-agent-loop to fix that. A "loop" is a small, repeatable workflow: something triggers Claude, it reads the current state, does one bounded action, checks its own work, saves what it learned, and either stops, retries, or flags a human. It's the difference between re-explaining a task every single day and setting it up once, correctly, with real guardrails.

13 patterns, one safe default

The skill ships with 13 ready-to-use loop patterns across different roles — engineering, DevOps, product, data, QA, security, support, marketing, HR, finance, legal, sales, and education — each with a copy-paste starter prompt. Every single one defaults to a low permission level: read, compare, draft, and flag for a human. None of them send an email, approve an expense, sign a contract, or touch a grade without a person clicking the button.

That constraint is the whole point. The goal was never to give Claude more authority — it was to take the boring, repetitive middle of a task off my plate so I get the time back. If a loop is doing its job, it should feel like reclaiming twenty minutes every morning, not like handing over control.

Pick one recurring check you do manually, hand it to a loop, and see what you get back.

How a loop actually works

Under the hood every loop is the same six-beat cycle, and it never self-certifies with "looks done":

  • Trigger — a timer, an event, or a /loop command starts a run.
  • Context — Claude reads a stable TASK.md and its own PROGRESS.md memory.
  • Action — it takes one constrained, in-scope step.
  • Verify — it runs a concrete pass/fail checklist, not a vibe.
  • State — it writes what happened back to PROGRESS.md.
  • Decide — stop, retry once, or escalate to a human.

Install it in under a minute

  1. Download build-agent-loop.skill from the button above.
  2. In claude.ai, go to Settings → Capabilities → Skills → Add → Upload a skill.
  3. Ask Claude to "set up a loop that…", or copy one of the 13 starter prompts.

If you're already using Claude day to day, this is worth trying. Start with a single recurring task, keep the permission level low, and let it earn trust one run at a time.
